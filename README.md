# XPGame — Your life as a single-player RPG

> **Set goals. Complete tasks. Earn XP. Level up. Spend XP on rewards you actually want.**

XPGame is a Telegram mini-app that turns personal development into a video game. Drop in a few raw thoughts about what you want from your life — the AI coach (GPT-5) maps them into goals, generates a personal roadmap, and breaks it down into time-boxed tasks calibrated to your level. You finish a task → you earn XP → you level up → new mechanics unlock → you spend the XP in a shop *you* designed (movie night? expensive coffee? game time?). The coach watches what you actually do versus what you planned, and quietly reshapes the roadmap as you go.

## Why

Most habit and productivity apps treat self-improvement as a static checklist. XPGame treats it as a single-player RPG. Every meaningful action awards XP. Levels unlock new mechanics. The coach behind the scenes constantly reshapes your roadmap based on what you *actually* do, not what you planned six weeks ago.

## What's inside

### 🤖 An AI coach that knows you
- **Goal & weakness analysis** — drop in raw thoughts, the coach returns a structured map of objectives plus the friction stopping you
- **Personalised roadmaps** generated by GPT-5, calibrated to your level
- **Conversational continuity** via the OpenAI Responses API — the coach remembers context across sessions
- **Auto-replanning** when you complete or stall on tasks
- **Daily motivation** based on your real progress (cached to keep token spend low)
- **Onboarding chat** — guided first conversation that turns `/start` into a populated profile

### 🎮 A real game economy
```js
calculateXP(difficulty)     // easy = 15, medium = 35, hard = 75
calculateLevel(totalXP)     // floor(sqrt(totalXP / 100)) + 1
getXPForNextLevel(level)    // level^2 * 100
```

- 🏆 **Achievements** unlock based on combinations of actions
- 📜 **XP history** — every grant, deduction, and reason is auditable
- 🛒 **A shop you designed** — boosters (temporary XP multipliers) and rewards (one-shot or duration-based perks)
- ⚡ **Active boosters** track timed effects per user

### ✅ Productivity that doesn't feel like productivity
- **Goals** with priority and category
- **Roadmap tasks** with dependencies and refresh tracking
- **Manual tasks** for things the coach didn't suggest
- **Today** view — what's due now
- **Budget tracker** — real-world spending alongside the XP economy
- **Leaderboard** — friendly competition across users on the same instance

## Architecture

```

│  Telegram WebApp        │  HTTPS  │  Express server                │
│  (React + Vite)         │ ───────▶│  (server/index.js, port 3007)  │
│                         │         │  ┌─────────────────────────┐   │
│  17 React components    │◀─────── │  │ • Telegram bot (grammy) │   │
│  in /src/components     │         │  │ • OpenAI Responses API  │   │
└─────────────────────────┘         │  │ • PostgreSQL pool       │   │
                                    │  │ • 26 REST endpoints     │   │
                                    │  └─────────────────────────┘   │
                                    └───────────────┬───────────────┘
                                                    │
                                                    ▼
                                          ┌────────────────────┐
                                          │  PostgreSQL        │
                                          │  8 base + 3 shop   │
                                          │  tables            │
                                          └────────────────────┘
```

| Folder | What it is |
| --- | --- |
| `src/` | The React + Vite mini-app — Dashboard, Today, Tasks, Goals, OnboardingChat, Shop, Profile, Achievements, Leaderboard, Budget, UnifiedChat, CountdownTimer |
| `server/` | Express bootstrap, grammy bot, gamification engine, OpenAI service, shop, models, 26 REST routes |
| `server/migrations/` | 12 forward-only migrations tracking the platform's evolution |
| `server/schema.sql` | Initial schema — users, goals, weaknesses, tasks, achievements, conversations, xp_history |

## Run it locally

Prerequisites: Node 20+, PostgreSQL 14+, a Telegram bot token from [@BotFather](https://t.me/BotFather), an OpenAI API key.

```bash
# 1. Database
createdb xpgame_db
cd server
node init-database.js          # runs schema.sql + every migration in order

# 2. Backend
cp .env.example .env           # fill in TELEGRAM_BOT_TOKEN, WEBAPP_URL,
                               # OPENAI_API_KEY, DB_*
npm install
npm start                      # Express on :3007

# 3. Frontend (repo root)
cd ..
npm install
npm run dev                    # Vite on :5173
```

For production: `npm run build` produces a `dist/` that's served as static assets behind Caddy at the same origin as the API.

## Configuration

The full list lives in [`server/.env.example`](server/.env.example):

- `TELEGRAM_BOT_TOKEN` + `WEBAPP_URL` — required
- `OPENAI_API_KEY` — required for AI features (manual gameplay still works without it)
- `OPENAI_PLAN_MODEL` — defaults to `gpt-5`
- `DB_HOST`, `DB_PORT`, `DB_NAME`, `DB_USER`, `DB_PASSWORD`
- `PORT`, `NODE_ENV`

## Maintenance scripts

Plain Node scripts in `server/` for one-shot operations:

```
init-database.js         Bootstrap a fresh DB
apply-shop-migration.js  Apply just the shop schema
check-shop-items.js      List current shop catalog
check-tables.js          Sanity-check that all tables exist
clean-duplicates.js      Dedupe tasks / goals
clear-rewards.js         Wipe shop_items and purchases
clear-users.js           Nuke all users (dev only!)
run-migration.js         Re-run the migration runner
```
